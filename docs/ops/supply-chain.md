# Supply-chain и статические проверки

## 1. Назначение
Документ описывает минимальные проверки целостности поставки (SBOM, уязвимости) и статические сканеры, доступные в репозитории. Проверки детерминированы и не требуют внешней инфраструктуры по умолчанию.

## 2. Команды и режимы

- `make sbom` — генерация SBOM для контейнерных образов (Syft).
- `make vuln-scan` — проверка образов на уязвимости (Grype).
- `make supply-chain` — последовательный запуск `sbom` и `vuln-scan`.

- `make sast-scan` — статический анализ исходного кода (SAST), **env‑gated**:
  - запускается только при `ANIMUS_SAST_SCAN=1`;
  - при отсутствии `gosec` возвращает ошибку (без сетевой установки по умолчанию);
  - для автоматической установки разрешите `ANIMUS_SAST_INSTALL=1`.

- `make dep-scan` — сканирование зависимостей контейнерных образов, **env‑gated**:
  - запускается только при `ANIMUS_DEP_SCAN=1`;
  - при отсутствии `grype` возвращает ошибку (без сетевой установки по умолчанию);
  - для автоматической установки разрешите `ANIMUS_DEP_SCAN_INSTALL=1`.

- `make openapi-compat` — проверка совместимости OpenAPI относительно базовой «снимковой» версии:
  - базовая версия хранится в `open/api/baseline/`;
  - при обнаружении различий проверка завершается ошибкой;
  - обновление базовой версии выполняется с `OPENAPI_BASELINE_UPDATE=1`.

## 3. Артефакты и хранение
По умолчанию результаты сохраняются в `.cache/supply-chain/` и не добавляются в репозиторий. При необходимости путь можно переопределить через переменные окружения, указанные в скриптах `scripts/*.sh`.
